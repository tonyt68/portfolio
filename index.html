How do you revoke a user's access the instant IT disables them — and guarantee a fresh login can't quietly undo it?

  • Federation Hub, Not a Point Integration: the post-login Action's group → role → action mapping is keyed by connection name — onboarding a second IdP (an acquired company's own Okta, Azure AD, PingFederate) is a new Auth0 Enterprise Connection plus a new entry in the same Action, never a resource-server code change. Nothing native in Auth0 computes this claim from SAML group attributes on its own, either — it's hand-written, not a dashboard toggle.
